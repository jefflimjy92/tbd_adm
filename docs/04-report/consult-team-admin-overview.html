상담팀 실무 화면 안내

상담팀 실무자가 실제로 쓰는 화면을 기준으로, “어느 화면에서 무엇을 확인하고 무엇을 입력하는지”를 쉽게 이해할 수 있게 다시 정리한 문서입니다. 개발 용어는 최대한 빼고, 실제 업무 흐름 중심으로 설명했습니다.

누가 어떤 화면을 쓰는지

실무상 상담팀원과 상담팀장이 쓰는 화면이 조금 다릅니다.

상담팀원

실무 입력 중심

상담팀원은 본인에게 배정된 고객을 보고 상담 내용을 입력하고, 인계가 필요한 고객은 영업팀으로 넘길 수 있도록 정보를 정리하는 역할에 맞춰 화면이 구성되어 있습니다.

대시보드 처리 현황 상담/TM 일일 보고서
자기 담당 건 위주로 확인 상담 결과 입력 중심 직원 현황은 본인 건만 보이는 구조

상담팀장

배정/관리 추가

상담팀장은 상담팀원이 쓰는 기본 화면에 더해, 미배정 건 배분, 직원별 진행 현황, 품질/피드백 확인 같은 관리용 화면을 더 볼 수 있습니다.

대시보드 처리 현황 DB 분류/배정 상담/TM 일일 보고서
미배정 건 균등 배분 가능 직원별 진행 현황 확인 가능 부재/취소/품질 관리 가능

상담팀 실무 흐름

하루 업무를 순서대로 보면 아래 흐름으로 이해하면 가장 쉽습니다.

1

오늘 상황 먼저 보기

조회가 잘 되고 있는지, 신청 단계에서 많이 빠지는 구간이 어디인지 먼저 확인합니다.

2

접수 건 확인

현재 상담 단계에 있는 고객이 누구인지, 누가 담당 중인지, 어디서 멈췄는지 확인합니다.

3

DB 배정

팀장은 미배정 건을 상담원에게 나눠줍니다. 소개 건은 일반 건과 조금 다르게 봐야 합니다.

4

1차 상담

보험 상태, 건강 이력, 고객 반응을 보며 계속 진행 가능한 고객인지 판단합니다.

5

2차 상담

미팅으로 넘길지, 보류할지, 취소할지 정리하고 고객 성향과 주의사항을 남깁니다.

6

영업팀 인계

일정, 예상 환급금, 수수료 안내, 전달 메모까지 정리한 뒤 영업팀에 넘깁니다.

화면별 설명

각 화면을 실무자가 어떤 용도로 쓰는지 기준으로 정리했습니다.

대시보드

하루 시작 전에 전체 흐름을 보는 화면

현재 사용 가능
상담팀원/팀장 공통 현황 확인용
  • 조회 성공/실패, 신청 이탈률, 상담 품질을 한 번에 볼 수 있습니다.
  • 팀장 입장에서는 누가 콜을 많이 했는지, 연결률과 미팅 전환이 어떤지 보기 좋습니다.
  • 실제 입력 화면이라기보다 현재 상황을 파악하는 용도입니다.

처리 현황

고객이 현재 어느 단계에 있는지 보는 기본 화면

현재 사용 가능
상담팀원/팀장 공통 상황 확인용
  • 접수, 상담, 미팅, 청구, 종결 단계로 나눠 볼 수 있습니다.
  • 고객명, 담당자, 현재 상태, 어느 팀에서 잡고 있는지를 바로 확인할 수 있습니다.
  • 상세로 들어가면 중간에 어디서 멈췄는지 파악하기 쉽습니다.

DB 분류/배정

팀장이 미배정 건을 나눠주는 화면

현재 사용 가능
상담팀장 중심 배정 업무용
  • 미배정, 일반DB, 소개DB를 나눠서 볼 수 있습니다.
  • 고객 나이, 지역, 동의 여부를 보며 배정 판단을 할 수 있습니다.
  • 여러 건을 묶어 상담원들에게 나눠줄 수 있습니다.
  • 다만 자동 배정이 아니라 팀장이 확인 후 확정하는 방식입니다.

상담/TM 목록

상담팀이 가장 자주 쓰는 메인 화면

현재 사용 가능
상담팀원/팀장 공통 건별 처리 + 직원 현황
  • 오늘 상담할 고객 목록을 건별로 보고 바로 상세로 들어갈 수 있습니다.
  • 직원 현황 보기로 바꾸면 상담원별 진행 상태를 한 번에 볼 수 있습니다.
  • 팀원은 자기 건 중심, 팀장은 전체 인원 현황을 보는 구조입니다.

상담 상세 처리

실제 상담 내용을 입력하는 핵심 화면

현재 사용 가능
상담팀원/팀장 공통 실제 입력 화면
  • 1차 상담, 2차 상담, 최종 결과를 단계별로 선택할 수 있습니다.
  • 보험 상태, 건강 이력, 고객 성향, 가족/지인 여부, 동반 신청 등을 같이 정리할 수 있습니다.
  • 미팅으로 넘길 고객이면 예상 환급금, 수수료 안내 여부, 일정, 전달 메모까지 적을 수 있습니다.
  • 스크립트가 같이 떠서 상담하면서 바로 확인하기 좋습니다.

1차 상담 전용 / 2차 상담 전용 / 인계 체크 전용 화면

메인 상담 화면을 단계별로 나눠서 보는 보조 화면

보조 화면
단계별 보기 필요 시 활용
  • 실무상 메인 상담 화면 하나로 대부분 처리 가능하지만, 상황에 따라 단계별로만 보고 싶을 때 쓸 수 있습니다.
  • 1차 상담만 모아 보기, 2차 상담만 모아 보기, 인계 체크만 집중해서 보기에 적합합니다.
  • 기본 동선은 메인 상담 화면 중심입니다.

케이스 상세

한 고객을 팀 전체 기준으로 보는 화면

현재 사용 가능
상담팀원/팀장 공통 팀간 연결 확인용
  • 상담팀, 영업팀, 청구팀 흐름을 한 화면 안에서 이어서 볼 수 있습니다.
  • 상담팀 구간에서는 건강 체크, 보험 요약, 초기 반응, 심화 상담 메모, 미팅 적합도를 정리할 수 있습니다.
  • 다음 팀이 어떤 정보를 받게 되는지 확인하기 좋습니다.

인계 확인 팝업

영업팀으로 넘기기 전에 마지막으로 보는 확인 창

확인용 팝업
상담팀 → 영업팀 인계 직전
  • 건강 체크, 상담 입력, 고객 메모, 미팅 적합 여부가 빠지지 않았는지 확인합니다.
  • 필수 항목이 빠진 상태로도 넘길 수는 있지만 경고가 뜹니다.
  • 즉, “이 고객을 지금 넘겨도 되는가”를 마지막으로 걸러주는 장치입니다.

일일 보고서

중간 보고와 마감 보고를 정리하는 화면

현재 사용 가능
상담팀원/팀장 공통 보고/관리용
  • 인계 건수, 취소 사유, 부재 관리, 관리 고객, 스크립트 피드백을 나눠서 볼 수 있습니다.
  • 13시 중간 보고와 18시 30분 마감 보고를 나눠서 볼 수 있습니다.
  • 팀장 입장에서는 직원 관리와 품질 피드백 화면으로도 의미가 있습니다.

실무자가 특히 알아두면 좋은 점

왜 경고가 뜨는지, 왜 다음 단계로 안 넘어가는지 쉽게 이해할 수 있는 내용만 모았습니다.

기본 원칙

담당자별로 보는 범위가 다릅니다

  • 상담팀원은 자기 처리 중심 화면이 열립니다.
  • 상담팀장은 배정과 직원 현황까지 더 볼 수 있습니다.
  • 그래서 팀원과 팀장이 보는 목록 범위가 완전히 같지는 않습니다.
입력 누락 방지

필수 항목이 빠지면 다음으로 못 넘깁니다

  • 보험 정보, 건강 체크, 고객 성향 같은 핵심 정보가 비어 있으면 경고가 뜹니다.
  • 미팅 인계로 넘길 때는 일정, 수수료 안내, 녹취/자료 첨부 확인이 더 필요합니다.
  • 다음 팀이 다시 처음부터 묻지 않게 하려는 목적입니다.
즉시 확인

진행이 어려운 고객은 바로 표시됩니다

  • 보험이 아예 없는 경우
  • 보험이 미납 또는 실효 상태인 경우
  • 기존 설계사와 가족·지인 관계여서 진행이 어려운 경우
질병 경고

중대질환 관련 내용이 있으면 경고가 뜹니다

  • 암, 뇌, 심장 관련 표현이 들어가면 별도 확인이 필요하다는 안내가 뜹니다.
  • 상담원이 놓치지 않도록 팀장 검토가 필요한 신호로 쓰기 좋습니다.
정리 기준

부재/거절도 자동으로 경고합니다

  • 부재가 많이 쌓인 경우
  • 고객이 명확히 거절한 경우
  • 오랫동안 응답이 없는 경우
  • 상담을 더 끌지 말고 정리해야 하는 신호로 볼 수 있습니다.
다음 팀 전달용

인계 메모는 가장 중요한 정보입니다

  • 고객 반응, 결정권자, 선호 시간대, 주의사항이 빠지면 다음 팀이 바로 움직이기 어렵습니다.
  • 환급형 고객은 어떤 포인트를 더 봐야 하는지까지 적게 되어 있습니다.
  • 즉 인계 메모는 “다음 팀이 고객을 다시 처음부터 묻지 않게 하는 정보”입니다.

현재 실무에서 기대할 수 있는 수준

무조건 된다/안 된다보다, 실제로 쓰면 어디까지 가능한지 관점으로 정리했습니다.

업무 영역 지금 가능한 수준 실무자가 느끼는 장점 아직 부족한 점
조회/신청 모니터링 기초 현황 확인 가능 오늘 조회가 잘 되는지, 신청 이탈이 어디서 많은지 빠르게 볼 수 있습니다. 외부 조회 시스템 상태를 깊게 파고드는 전용 화면은 없습니다.
접수 건 확인 기본 흐름 확인 가능 지금 누가 어디까지 처리했는지 보기에 충분합니다. 실제 운영 데이터가 계속 쌓이는 구조는 아직 아닙니다.
DB 배정 수동 배정 중심 미배정 건을 골라 상담원에게 나누는 데는 바로 쓸 수 있습니다. 자동으로 시간대별/잔여량별 배정해주는 수준은 아닙니다.
1차 상담 실제 입력 가능 필수 확인 항목을 보면서 상담 결과를 정리하기 좋습니다. 전화 발신, 자동 녹취 업로드까지 바로 되는 구조는 아닙니다.
2차 상담 실제 입력 가능 미팅으로 넘길지, 보류할지, 취소할지 정리하기 좋습니다. 실시간 코칭이나 자동 통화 품질 분석 수준은 아닙니다.
영업팀 인계 실무 메모 정리 가능 영업팀에 넘기기 전 필요한 내용을 한 화면에서 정리할 수 있습니다. 실제 자동 문자 발송이나 외부 시스템 전송까지는 아닙니다.
보고/품질 관리 운영 참고용으로는 충분 마감 보고, 부재/취소 관리, 스크립트 피드백 화면이 있어 팀장 관리에 도움이 됩니다. 자동 집계 데이터라기보다 운영 참고용 화면 성격이 더 강합니다.
단계별 전용 화면 보조용으로 존재 상황에 따라 1차/2차/인계만 따로 보고 싶을 때 활용 여지가 있습니다. 기본 동선은 메인 상담 화면 하나로 보는 쪽에 가깝습니다.

실무자 기준 추천 동선

처음 보는 사람도 아래 순서대로 보면 흐름을 이해하기 쉽습니다.

상담팀원

하루 업무 순서

  • 처리 현황에서 오늘 볼 고객을 먼저 확인
  • 상담 화면에서 1차/2차 상담 입력
  • 넘길 고객은 일정과 전달 메모까지 적고 인계
  • 마감 전에는 일일 보고서에서 인계/부재/취소 현황 확인
상담팀장

관리 업무 순서

  • DB 분류/배정에서 미배정 건을 정리
  • 상담 화면의 직원 현황으로 인원별 진행 상태 확인
  • 일일 보고서에서 부재, 취소, 품질, 피드백을 종합 관리
  • 필요하면 케이스 상세에서 영업팀에 넘기기 전 상태를 다시 점검
실무 관점 요약. 지금 수준은 상담팀 흐름을 한 번에 정리하고 입력 누락을 줄이는 데는 충분한 화면입니다. 다만 실제 운영 시스템처럼 전화, 알림톡, 외부 조회, 자동 저장까지 완전히 붙어 있는 단계는 아닙니다. 따라서 이 문서는 “현재 화면으로 무엇을 할 수 있는지”를 이해하는 기준으로 보는 것이 가장 적절합니다.
