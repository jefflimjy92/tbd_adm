01

상담팀 실무 화면 캡처 자료

실제 화면을 기준으로 상담팀이 어떤 순서로 일하는지, 각 화면에서 무엇을 확인하고 입력하는지, 그리고 다음 팀에 무엇이 넘어가는지 한 장씩 설명한 슬라이드형 자료입니다.

이 자료는 이렇게 보면 됩니다

  • 슬라이드 오른쪽은 실제 화면 캡처입니다.
  • 왼쪽은 실무자가 이 화면에서 하는 일을 쉬운 말로 정리한 설명입니다.
  • 개발 용어보다 업무 흐름 중심으로 읽으면 됩니다.
주요 화면
8

6단계
주요 대상
상담팀원
관리 대상
상담팀장

기본 흐름

Step 1

상황 확인

조회/신청 현황을 먼저 봅니다.

Step 2

접수 건 확인

오늘 처리할 고객과 현재 단계를 확인합니다.

Step 3

DB 배정

팀장은 미배정 건을 상담원에게 나눕니다.

Step 4

1차 상담

보험 상태와 건강 이력을 확인합니다.

Step 5

2차 상담

미팅 가능 여부와 주의사항을 정리합니다.

Step 6

영업팀 인계

일정과 전달 메모를 정리해 넘깁니다.

02

대시보드

하루 시작 전에 조회, 신청, 상담 품질 상태를 한 번에 보는 화면입니다.

이 화면에서 하는 일

  • 조회 성공/실패가 어떤지 확인
  • 신청 단계에서 어디서 많이 빠지는지 확인
  • 상담원별 콜 수, 연결률, 미팅 전환, 품질 점수 확인

실무 포인트

  • 상담팀원보다는 팀장이 더 자주 보게 되는 화면입니다.
  • 오늘 조회가 흔들리는지, 누가 도움 필요한지 빠르게 파악하기 좋습니다.

03

처리 현황

고객이 지금 어디 단계에 있는지, 누가 잡고 있는지 보는 기본 화면입니다.

이 화면에서 하는 일

  • 접수, 상담, 미팅, 청구, 종결 단계별로 고객을 나눠봅니다.
  • 고객별 현재 상태와 담당자, 팀 위치를 확인합니다.
  • 중간에 멈춘 건이나 오래 걸리는 건을 찾기 좋습니다.

실무 포인트

  • “오늘 내가 봐야 할 고객이 누구인가”를 정리하기 좋습니다.
  • 다른 팀으로 넘어간 고객도 어느 단계까지 갔는지 같이 볼 수 있습니다.

04

DB 분류/배정

상담팀장이 미배정 고객을 상담원에게 나눠주는 화면입니다.

이 화면에서 하는 일

  • 미배정, 일반DB, 소개DB를 나눠서 확인
  • 나이, 지역, 동의 여부를 보고 배정 판단
  • 여러 건을 한 번에 선택해서 상담원들에게 나눠주기

실무 포인트

  • 자동 배정보다 팀장이 보고 확정하는 성격이 강합니다.
  • 소개 건은 일반 건과 다른 흐름으로 봐야 하므로 따로 확인이 필요합니다.

05

상담/TM 목록

상담팀이 가장 많이 쓰는 메인 화면으로, 건별 목록과 직원 현황을 오가며 봅니다.

이 화면에서 하는 일

  • 오늘 상담할 고객 목록을 건별로 확인
  • 필요한 고객을 바로 상세 화면으로 열기
  • 직원 현황 보기로 바꿔 상담원별 진행 상태 확인

실무 포인트

  • 팀원은 자기 건 중심으로 보고, 팀장은 전체 상황을 보게 됩니다.
  • “누가 어디까지 진행했는지” 확인하기에 가장 실무적인 화면입니다.

06

상담 상세 입력

보험 정보, 건강 이력, 고객 성향, 일정, 전달 메모까지 실제 상담 내용을 정리하는 핵심 화면입니다.

이 화면에서 하는 일

  • 1차 상담, 2차 상담, 최종 결과를 단계별로 선택
  • 보험 상태, 건강 이력, 가족/지인 여부, 동반 신청 여부 입력
  • 미팅으로 넘길 고객이면 일정, 예상 환급금, 수수료 안내 여부, 전달 메모까지 정리

실무 포인트

  • 입력 누락이 있으면 다음 단계로 잘 안 넘어가도록 되어 있습니다.
  • 진행 불가 사유나 주의가 필요한 고객은 화면에서 바로 경고가 뜹니다.

07

상담 스크립트 화면

실제 상담 멘트를 보면서 응대할 수 있도록 스크립트가 같이 열리는 화면입니다.

이 화면에서 하는 일

  • 1차 상담 스크립트와 2차 상담 스크립트를 구분해서 확인
  • 질문 항목에 맞춰 입력 화면으로 바로 이동
  • 주의사항과 활용 가이드를 함께 확인

실무 포인트

  • 신입이나 수습 인원에게 특히 도움이 되는 화면입니다.
  • 실제 업무에서는 입력 화면과 같이 열어놓고 쓰는 느낌에 가깝습니다.
스크립트 참고 신입 교육용으로도 유용

08

케이스 상세

한 고객을 상담팀, 영업팀, 청구팀 흐름까지 이어서 볼 수 있는 화면입니다.

이 화면에서 하는 일

  • 한 고객이 지금 어느 팀에 걸쳐 있는지 한눈에 확인
  • 상담팀 구간에서 건강 체크, 보험 요약, 심화 상담 메모를 다시 확인
  • 다음 팀이 어떤 정보를 받는지 미리 점검

실무 포인트

  • 단순 상담 입력 화면보다 “팀 간 연결”을 이해하기 쉬운 화면입니다.
  • 고객 하나를 기준으로 전체 흐름을 설명할 때 유용합니다.

09

인계 확인 팝업

영업팀으로 넘기기 전에 필수 항목이 빠지지 않았는지 마지막으로 확인하는 창입니다.

이 화면에서 하는 일

  • 건강 체크, 상담 입력, 고객 메모, 미팅 적합 여부 확인
  • 필수 누락이 있으면 경고를 보고 다시 보완
  • 그래도 넘겨야 하면 누락 인계로 기록 가능

실무 포인트

  • 영업팀이 받았을 때 “정보가 비어 있다”는 문제를 줄이는 역할입니다.
  • 인계 품질을 맞추는 마지막 관문이라고 보면 됩니다.

10

일일 보고서

중간 보고와 마감 보고를 정리하고, 부재/취소/품질을 같이 관리하는 화면입니다.

이 화면에서 하는 일

  • 인계 건수, 취소 사유, 부재 고객, 관리 고객 현황 확인
  • 스크립트 피드백, 품질 피드백, 수습 평가 내용 확인
  • 13시 중간 보고와 18시 30분 마감 보고를 구분해서 보기

실무 포인트

  • 상담팀원은 마감 참고용, 팀장은 관리용으로 더 많이 쓰게 됩니다.
  • 부재와 취소가 누적되는 인원을 찾기에 유용합니다.

11

한 줄로 정리하면

지금 화면 구성은 상담팀이 “누가 무엇을 보고, 어디까지 입력했고, 다음 팀에 무엇을 넘겨야 하는지”를 정리하는 데 충분한 수준입니다.

상담팀원 입장

  • 처리 현황에서 오늘 볼 고객 확인
  • 상담 상세에서 1차/2차 상담 입력
  • 넘길 고객은 일정과 전달 메모까지 정리
  • 마감 전에 일일 보고서로 인계/부재/취소 확인

상담팀장 입장

  • DB 분류/배정에서 미배정 건 정리
  • 상담 목록의 직원 현황으로 팀 상황 확인
  • 일일 보고서에서 부재, 취소, 품질, 피드백 종합 관리
  • 필요 시 케이스 상세에서 인계 전 상태 재확인
